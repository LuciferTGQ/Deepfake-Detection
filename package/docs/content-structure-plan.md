# 内容结构规划 - Deepfake Detection 智能检测AI换脸

## 1. 素材清单

**内容文件:**
- 无外部文档（所有文案将内嵌于UI中）

**视觉资产:**
- 无预置图片（需使用图标库提供的SVG图标）
- 背景将使用CSS渐变实现

**数据文件:**
- 无外部数据文件（检测结果为实时生成）

## 2. 网站结构

**类型:** SPA（单页应用）- 两个视图状态切换

**原因:** 
- 仅2个界面视图（欢迎页 → 主应用界面）
- 简单的流程（确认协议 → 使用应用）
- 无复杂内容结构，适合状态切换
- 符合现代Web应用体验

## 3. 页面/视图分解

### 视图 1: 欢迎页面 (`/#welcome`)

**目的:** 用户首次访问时的引导和协议确认

**内容映射:**

| 区块 | 组件模式 | 数据源 | 提取内容 | 视觉资产 |
|------|----------|--------|----------|----------|
| 顶部标题区 | Hero Pattern（简化版） | 内嵌文案 | "智能检测AI换脸" + 副标题 | - |
| 使用说明区 | 内容卡片 | 内嵌文案 | Deepfake检测说明、使用步骤 | 图标：info-circle (Lucide) |
| 协议确认区 | 表单组件（勾选框） | 内嵌文案 | 用户协议文本 + 勾选框 | 图标：checkbox (Lucide) |
| 确认按钮区 | 主要CTA按钮 | - | "确认使用"按钮 | 图标：arrow-right (Lucide) |

**禁止内容（设计规范中决定）:**
- ❌ 布局样式（居中、间距、颜色）
- ❌ 视觉处理（玻璃效果、阴影、圆角）
- ❌ 交互状态（hover、disabled效果）

---

### 视图 2: 应用主界面 (`/#app`)

**目的:** 视频上传、分析和结果展示

**内容映射:**

| 区块 | 组件模式 | 数据源 | 提取内容 | 视觉资产 |
|------|----------|--------|----------|----------|
| 顶部标题栏 | 导航栏（简化版） | 内嵌文案 | "智能检测AI换脸" 标题 | Logo图标：shield-check (Lucide) |
| 上传区域 | 文件上传组件 | 内嵌文案 | "上传视频（不超过500MB）" + 提示文字 | 图标：upload-cloud (Lucide), video (Lucide) |
| 分析按钮区 | 主要CTA按钮（状态控制） | - | "开始分析"按钮 | 图标：play-circle (Lucide) |
| 进度展示区 | 进度条组件 | 实时数据 | 分析进度百分比 | 图标：loader (Lucide, 旋转动画) |
| 结果展示区 | 结果卡片 | 实时数据 | 信任度百分比 + 评估结果文字 | 图标：check-circle (成功), alert-triangle (警告), x-circle (危险) |

**状态说明:**
- 初始状态：上传区可见，分析按钮禁用
- 上传后：分析按钮启用，显示文件信息
- 分析中：进度条可见，其他控件禁用
- 完成后：结果卡片显示，可重新上传

## 4. 内容分析

**信息密度:** 低
- 总内容量：约300-400字说明文字
- 交互元素：6个主要组件（按钮、上传区、进度条、结果卡片等）
- 视觉焦点：明确单一（欢迎页→确认按钮，主界面→上传区）

**内容平衡:**
- 图标：8-10个SVG图标（来自Lucide库）
- 数据/图表：1个进度条、1个百分比展示
- 文字：约400字（80%）
- 交互组件：20%

**内容类型:** 交互驱动型
- 主要目的：引导用户完成上传和查看结果
- 视觉简洁，功能明确
- 适合玻璃态设计的轻量级界面
